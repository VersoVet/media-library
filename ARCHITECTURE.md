## Module Responsibilities

### catalog
- Upload media files (multipart)
- Store in Dropbox + SQLite catalog
- Auto-extract metadata (dimensions, EXIF, duration, codec)
- CRUD + tag management (N:N relationship)

### dropbox
- Dropbox API wrapper (list, download, upload)
- Vault token management
- Temporary link generation for serving

### tagger
- LLM vision analysis via Groq API (llama-4-scout-17b-16e-instruct)
- Base64 image encoding + multimodal prompting
- Fallback to metadata-based tags if vision fails
- Autonomous tag suggestions for imported media

### thumbnails
- Generate webp thumbnails (320px width, ratio maintained)
- Images: Pillow resize
- Videos: ffmpeg frame extraction + Pillow resize
- Lazy generation on first access

### search
- FTS5 SQLite full-text index (title, description, tags)
- Tag-based navigation with counts
- Pagination support

### sources
- CRUD for scan sources (Dropbox/Local/SSH)
- Persistent configuration
- Enable/disable toggle

### scanner
- Multi-source orchestration
- Dropbox: list_folder with pagination
- Local: pathlib.Path.rglob recursive scan
- SSH: asyncssh SFTP download
- Pipeline: download → extract metadata → upload Dropbox → catalog → suggest tags → generate thumbnail

## Database Schema

```sql
CREATE TABLE media (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    description TEXT DEFAULT '',
    media_type TEXT NOT NULL,       -- 'image' or 'video'
    mime_type TEXT NOT NULL,
    dropbox_path TEXT NOT NULL,     -- Primary storage location
    source_id INTEGER,              -- Origin source
    source_path TEXT,               -- Path in source
    file_size INTEGER,
    width INTEGER,
    height INTEGER,
    duration_seconds REAL,          -- Videos only
    metadata_json TEXT DEFAULT '{}', -- EXIF, codec, etc
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT UNIQUE NOT NULL
);

CREATE TABLE media_tags (
    media_id TEXT REFERENCES media(id) ON DELETE CASCADE,
    tag_id INTEGER REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (media_id, tag_id)
);

CREATE VIRTUAL TABLE media_fts USING fts5(
    title, description, tags,
    content='media',
    content_rowid='rowid'
);

CREATE TABLE scan_sources (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    source_type TEXT NOT NULL,     -- 'dropbox', 'local', 'ssh'
    config_json TEXT NOT NULL,     -- Type-specific config
    enabled INTEGER DEFAULT 1,
    recursive INTEGER DEFAULT 1,
    auto_tag INTEGER DEFAULT 1,
    cron_schedule TEXT DEFAULT '0 */6 * * *',
    last_scan_at TEXT,
    last_scan_status TEXT
);

CREATE TABLE scan_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_id INTEGER REFERENCES scan_sources(id),
    started_at TEXT NOT NULL,
    finished_at TEXT,
    files_found INTEGER DEFAULT 0,
    files_imported INTEGER DEFAULT 0,
    files_skipped INTEGER DEFAULT 0,
    errors_json TEXT DEFAULT '[]'
);
```

## Storage

- **Original Files**: Dropbox `/media-library/{id}.{ext}` (primary)
- **Thumbnails**: Local `/opt/onyx/data/media-library/thumbnails/{id}.webp` (cached)
- **Metadata**: SQLite local `/opt/onyx/data/media-library/media.db`

## Deployment

- **Host**: OnyxAxon (10.0.0.21)
- **Port**: 8202
- **LLM Router**: `http://localhost:8055` (Groq vision)
- **Vault**: `http://10.0.0.44:8050` (Dropbox token + SSH keys)
- **Service**: systemd `onyx-media-library`
